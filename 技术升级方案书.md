### 功能 ② 策略/Agent 系统

**目标**：定义 15 个交易策略为 YAML 知识库，Agent 按策略筛选股票并给出操作建议。

#### 2.1 策略 YAML 格式

```yaml
# backend/strategies/ma_golden_cross.yaml

name: 均线金叉
aliases: [MA金叉, Golden Cross, 金叉买入]
description: |
  短期均线从下方向上穿越长期均线，是典型的看涨信号。
  
judgment_criteria: |
  1. MA5 上穿 MA10 或 MA20
  2. 成交量至少是前5日均量的1.2倍（放量确认）
  3. MACD 处于零轴上方或即将金叉
  4. 股价位于 MA60 上方（中长期趋势保护）
  
scoring_adjustment: |
  - 金叉时成交量放大 1.5x 以上: +20 分
  - 同时符合 MA5上穿MA10 和 MA10上穿MA20（三线开花）: +30 分
  - MACD 负值区金叉（弱势反弹）: -20 分
  
tools_required:
  - get_realtime_quote
  - get_kline
  - compute_ma
  - compute_macd
```

#### 3.2 断路器模式

```python
# backend/app/services/data_provider/base.py (新建)

@dataclass
class CircuitBreaker:
    """断路器：连续失败 N 次后熔断，冷却后尝试半开"""
    failure_threshold: int = 3
    cooldown_seconds: float = 60.0
    half_open_timeout: float = 10.0
    
    failure_count: int = 0
    last_failure_time: float = 0
    state: str = "closed"  # closed | open | half_open
    
    async def call(self, fetcher, *args, **kwargs):
        if self.state == "open":
            if time.time() - self.last_failure_time > self.cooldown_seconds:
                self.state = "half_open"
            else:
                raise CircuitOpenError("断路器熔断中")
        try:
            result = await fetcher(*args, **kwargs)
            self.failure_count = 0
            self.state = "closed"
            return result
        except Exception:
            self.failure_count += 1
            self.last_failure_time = time.time()
            if self.failure_count >= self.failure_threshold:
                self.state = "open"
            raise
```

#### 7.1 告警规则模型

```python
class AlertRule:
    stock_code: str          # 股票代码
    alert_type: str          # price_threshold / indicator / news
    condition: str           # "price > 100" / "ma5_cross_ma10" / "volume_spike"
    value: float             # 阈值
    severity: str            # info / warning / critical
    notify_channels: list    # 推送渠道
    cooldown_minutes: int    # 冷却时间
```
